部署與資料庫規劃

既然選擇了分離架構與 Docker，我們需要決定資料庫的處理方式：

A

完全雲端託管 (推薦)

使用 Neon, Supabase (Postgres) 或 Railway 的資料庫。優點：部署最穩，不必擔心資料庫容器掛掉或磁碟空間。缺點：需註冊新平台。

B

全容器化 (Docker Compose)

將資料庫也放在 Docker 容器中。優點：環境完全一致，適合本地開發。缺點：上線到 Render/Railway 等平台時，通常需要額外設定 Persistent Volume 或直接使用平台的資料庫服務。

後續規劃：

  • 後端：編譯為 JS 後執行，移除動態 DB Sync 邏輯。
  • 前端：優化 API URL 配置，增加錯誤處理。
  • 自動化：建立 Dockerfile 與 CI/CD 流程。
