錯題管理系統 MVP 設計方案

1. 技術棧 (Tech Stack)

Next.js (前端)
Express (後端 API)
Prisma (資料庫工具)
PostgreSQL (資料庫)
Docker (本地開發環境)

2. 資料流向 (Data Flow)

使用者瀏覽器
  ↳ [前端 Vercel] 送出請求
    ↳ [後端 Render] 處理邏輯
      ↳ [Prisma] 溝通資料
        ↳ [雲端 Neon DB] 存取資料

3. 部署與運維策略

  • 自動化部署：推送程式碼到 GitHub 後，Vercel 與 Render 會自動偵測並更新。
  • 環境隔離：
    • 開發環境：使用 Docker 模擬資料庫。
    • 正式環境：連接雲端 Neon 資料庫。
  • 安全性：敏感資訊（資料庫密碼等）存放在環境變數中，不進入 GitHub。
